# v0.0.11

**Status:** released (2026-06-22)

## Target

**Actordock** — optional ops dashboard with E2B-like sandbox observability and local cluster control. Visual language: **Greek marble sculpture** (museum-calm, classical typography, bronze accents).

| Before | After |
|--------|-------|
| CLI / curl / SDK only | Browser: sandboxes, templates, volumes, snapshots; metrics, logs, terminal |

Not part of the E2B REST surface. **REST parity unchanged** (still 27/56 after [v0.0.10](v0.0.10.md)).

## Goal

```bash
./hack/install-local.sh
kubectl port-forward -n actordock svc/dashboard 3000:3000
open http://localhost:3000
# → marble-themed dashboard: sandbox list → detail (metrics, logs, terminal)
# → templates / volumes / snapshots; create & kill sandboxes from UI
```

## In scope

| Tier | Deliverable |
|------|-------------|
| **P0** | Settings, Overview, Sandbox list, Sandbox detail (Overview / Metrics / Logs) |
| **P1** | Templates, Volumes, Snapshots (read-only pages) |
| **P2** | Sandbox actions (create, kill, pause, resume, timeout, snapshot), Terminal tab, aggregate Monitoring, metric sparklines, log export |
| **Infra** | `dashboard/` (Go server + `web/` SPA); `manifests/dashboard/` (Secret-based deploy); default local install includes dashboard; Platform BFF proxy |

Work breakdown: [v0.0.11-work.md](v0.0.11-work.md) · Issues [#61](https://github.com/actordock/actordock/issues/61)–[#71](https://github.com/actordock/actordock/issues/71)

## Out of scope

- New Platform REST routes
- SSO / multi-user auth
- Template build pipeline ([#59](https://github.com/actordock/actordock/issues/59))
- Billing, API keys, teams, webhooks
- Server-side metrics history / alerting

## Architecture

```
Browser → dashboard/ (SPA + /api/platform/* proxy) → Platform REST (X-API-KEY)
Terminal tab → POST /sandboxes/:id/connect → Router/envd Connect
Platform / Router / envd — unchanged when dashboard not deployed
```
