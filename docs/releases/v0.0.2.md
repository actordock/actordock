# v0.0.2

**Status:** released (2026-06-19)

## Target

**Visibility** — Sandboxes are trackable objects, not fire-and-forget.

| Before | After |
|--------|-------|
| Create/kill only | Query by id, list actives, state in Redis |

## Goal

```python
s = Sandbox.create(template="base", secure=False)
assert Sandbox.get_info(s.sandbox_id).sandbox_id == s.sandbox_id
sandboxes = Sandbox.list()  # includes s
s.kill()
```

```bash
./hack/install-local.sh && ./hack/verify-local.sh
```

## In scope

| Area | Deliverable |
|------|-------------|
| Store | Redis: sandbox id → actor id, template, created_at, status |
| Platform | `GET /sandboxes/{id}`, `GET /sandboxes`, `GET /v2/sandboxes` (E2B SDK list) |
| Platform | Sync status from Substrate on read |

## Out of scope

- Timeout, scheduler, autoPause
- Filesystem, secure tokens
- Pagination, filters, metrics

## API

| Method | Path |
|--------|------|
| GET | `/sandboxes/{id}` |
| GET | `/sandboxes` |
| GET | `/v2/sandboxes` |

## Milestones

1. Redis in `actordock` namespace; Platform read/write
2. Create writes Redis record; delete removes it
3. GET returns running/pending aligned with actor
4. LIST returns non-terminated sandboxes
5. `./hack/verify-local.sh` covers get_info and list (E2E)
