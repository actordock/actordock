# v0.0.8 — Work breakdown

Implementation sub-problems for [v0.0.8](v0.0.8.md). Four new Platform REST operations with **real** envd / Router / runtime behavior (not metadata-only stubs).

**Full-field rule:** each shipped operation includes **all** OpenAPI request/response fields. See [e2b-rest-parity.md](e2b-rest-parity.md).

## WP1 — envd `process.Connect`

- Replace `Unimplemented` with PTY-backed interactive shell attach
- Bidirectional stream: stdin/stdout/stderr (and resize if in proto)
- Unit tests with httptest Connect client; attach `/bin/sh`, send input, read output
- Same h2c server path as `process.Start`

## WP2 — Router connect proxy

- Route connect/WebSocket (or Connect RPC) from client to envd backend for sandbox
- Resume sandbox if paused (reuse existing backend resolution)
- Unit/integration tests: proxy reaches envd Connect handler

## WP3 — Platform `POST /sandboxes/{id}/connect`

- Full E2B ConnectSandbox request/response fields
- Sandbox missing → **404**; resume + wait envd when paused
- Response must point at Router/envd endpoint that **works** with WP1–WP2
- Platform unit tests

## WP4 — Platform network persist + GET backfill

- `PUT /sandboxes/{sandboxID}/network` — full schema; persist on Redis sandbox record
- `GET /sandboxes/{id}` returns stored `network` / `allowInternetAccess`
- **404** / **400** as per OpenAPI
- Platform unit tests for round-trip

## WP5 — Router network enforcement

- Read sandbox network config (from Platform store or forwarded header/cache)
- When `allow_internet_access=false`, block sandbox egress through Router proxy (minimal: reject non-local upstream)
- When `true`, existing behavior unchanged
- Router tests for allow vs deny

## WP6 — runtime snapshot create

- `POST /sandboxes/{sandboxID}/snapshots` triggers real checkpoint (Runtime suspend/checkpoint API)
- Persist snapshot metadata in Redis: id, sandbox_id, actor_id, runtime version, timestamps, OpenAPI fields
- Sandbox must be running or pausable; **404** if missing
- Platform unit tests with fake runtime client

## WP7 — `GET /snapshots`

- List snapshot records from Redis (all sandboxes; filters per OpenAPI if any)
- **200** with array; entries appear after WP6 POST
- Platform unit tests

## WP8 — E2E, docs, parity

- E2E connect: attach or SDK path runs a command interactively
- E2E network: with internet disabled, outbound request fails (or enabled succeeds)
- E2E snapshot: POST then GET lists new entry
- Update `docs/user/quickstart.md`; parity ops #16, #18, #20, #21 → `done (v0.0.8)`
- Update [v0.0.8.md](v0.0.8.md) Done when
