# v0.0.1 — Work breakdown

Implementation sub-problems for [v0.0.1](v0.0.1.md). Tackle roughly in order; later items depend on earlier ones.

## WP1 — Repository scaffold

- Go module, `cmd/` layout (`platform`, `router`, `envd`)
- `Makefile`: build, test, fmt
- `.ko.yaml` for container images
- Minimal `internal/` packages (config, logging)

## WP2 — Substrate deploy bundle

- `substrate.lock` (pinned commit / image tags)
- `hack/install-local.sh`: Kind cluster `actordock`, deploy Substrate, wait **valkey before ate-api**
- Manifests: `WorkerPool`, `ActorTemplate` stubs (ko resolve paths)
- `--skip-substrate` for redeploy-only Actordock

## WP3 — envd daemon

- HTTP `/health`, `/init`
- ConnectRPC `process.Start` (foreground, capture stdout/stderr)
- Listen on :49983 inside worker
- Container image build via ko

## WP4 — Template `base`

- `ActorTemplate` manifest pointing at envd image
- Golden snapshot reaches `Ready`
- Document: only `template_id=base` supported

## WP5 — Substrate gRPC client

- `internal/substrate`: dial ateapi, TLS for Kind
- `CreateActor`, `DeleteActor`, `ResumeActor`
- Map sandbox id ↔ actor id (same id for v0.0.1)

## WP6 — Platform API

- `GET /health`
- `POST /sandboxes` → `CreateActor`, return E2B-shaped JSON
- `DELETE /sandboxes/{id}` → delete actor
- Dev API key; `secure=false` → empty `envd_access_token`
- K8s Deployment + Service

## WP7 — Router

- Parse `{sandboxId}.{domain}` from Host
- `ResumeActor` before proxy
- Reverse proxy to worker IP:49983
- K8s Deployment + Service; port-forward :8081 locally

## WP8 — Local manifests

- `manifests/local/`: platform, router, kustomization
- ConfigMap: ateapi address, domain, dev API key
- `install-local.sh` applies Actordock after Substrate

## WP9 — End-to-end verify

- `examples/verify.py` + `requirements.txt`
- `hack/verify-local.sh`
- `hack/.env.local` generated by install script
- Port-forwards :8080 (platform), :8081 (router)

## WP10 — CI

- GitHub Actions workflow on PR / push to `main`
- `go test ./...`, `go vet`, fmt check
- Build images with ko (no push) or compile-only
- Optional: Kind smoke job (install + verify) — mark as manual/nightly if too heavy for every PR

## WP11 — User documentation

- `docs/user/quickstart.md`: prerequisites, install, env vars, run verify
- README links to quickstart (keep README short)
- Troubleshooting: 3–5 bullets (substrate not ready, port-forward, template not Ready)
