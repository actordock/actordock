# v0.1.1

**Status:** released

## Target

**Pilot + monorepo runtime + template builds** — v0.1.0 E2B pilot scope, Substrate merged into `runtime/`, custom template build pipeline, and naming cleanup.

| Before (v0.0.11) | After (v0.1.1) |
|------------------|----------------|
| External Substrate dependency | Vendored `runtime/` in monorepo |
| No template builds | v3 upload → Kaniko build → tagged ActorTemplates |
| 27/56 REST ops | ≥33 ops + ≥60% field parity (pilot) |

## Quick start

```bash
./hack/install-local.sh
./hack/verify-local.sh
```

Helm:

```bash
helm install actordock ./charts/actordock-stack -n actordock --create-namespace
```

## Highlights

### Monorepo & runtime

- Migrate Substrate into `actordock/runtime/` (runtime-api, runtime-worker, runtime-sandbox, …)
- Remove `ate` / `substrate` naming; Actordock + `runtime/` single repo
- Architecture diagram and component table in README

### v0.1.0 pilot (E2B REST + envd)

- envd filesystem RPC + HTTP `/files`
- `secure=true` sandbox create with envd access tokens
- Sandbox create/get/list field parity; api-keys and access-tokens routes
- `actordock-stack` Helm chart; v0.1.0 E2E parity tests

### Template builds (#91–#96)

- Redis build store; `POST /v3/templates` file upload
- `POST /v2/templates/{id}/builds/{buildID}` trigger + queue
- Kaniko template-builder worker; build status/logs/tags
- Official `python` base template; E2E template build test

### Examples & CI

- LlamaIndex policy Q&A example (`examples/llamaindex/`) in `verify-examples.sh`
- Dead migration code removed; CI example fixes for `base` template
