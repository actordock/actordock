# v0.0.8

**Status:** planned

## Target

**Volumes** — E2B volume CRUD on Platform.

| Before | After |
|--------|-------|
| No volume API | Create/list/get/delete volumes via REST |

## Goal

```python
from e2b import Volume  # or REST client
# list/create/get/delete volume — no 404 on Platform routes
```

## In scope

| Area | Deliverable |
|------|-------------|
| Platform | Full volume CRUD: `GET/POST /volumes`, `GET/DELETE /volumes/{id}` |
| Store | Volume metadata in Redis (or minimal in-memory for local) |
| Unit tests | CRUD round-trip |

## Out of scope

- Mount volumes into sandboxes at runtime
- GCS/S3 backing (metadata-only OK for v0.0.8)
- Template file APIs

## API

| Method | Path |
|--------|------|
| GET | `/volumes` |
| POST | `/volumes` |
| GET | `/volumes/{volumeID}` |
| DELETE | `/volumes/{volumeID}` |

**REST parity:** +4 ops → 22/56 (39%). See [e2b-rest-parity.md](e2b-rest-parity.md).
