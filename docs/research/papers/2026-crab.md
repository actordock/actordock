# Crab (2026)

**Citation:** Wu et al. *Crab: A Semantics-Aware Checkpoint/Restore Runtime for Agent Sandboxes.* arXiv, 2026.  
https://arxiv.org/html/2604.28138v1

## Problem they solve

Full per-turn sandbox C/R is too expensive under dense co-location; chat-only recovery misses OS effects. Bridge the agent–OS semantic gap to checkpoint only when recovery-relevant.

## Method

- eBPF inspector classifies turn effects → checkpoint granularity (none / FS / process / full).
- Coordinator aligns C/R with turn boundaries; overlaps with LLM wait.
- Host C/R engine schedules checkpoint jobs; promotes jobs whose LLM response already returned (exposed latency).

## Reuse for us

- Host-scoped C/R scheduling under density.
- “Hide C/R behind idle/LLM wait” as an optional scoring feature.
- Stress setting: many sandboxes/host, FIFO vs reactive C/R queue as a **micro-baseline** (for I/O, not slot allocation).

## Does not transfer

- Their “semantics” ≠ sandbox **priority classes** or cross-Worker placement/eviction.
- Optimizes **when/how to checkpoint**, not **which sandbox keeps the scarce Worker slot**.
