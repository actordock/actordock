# v0.0.5

**Status:** released (2026-06-20)

## Target

**Idle suspend** — Sandboxes can sleep instead of die; wake on next use.

| Before | After |
|--------|-------|
| Timeout always kills | `on_timeout=pause` suspends actor; next command resumes |

Uses Runtime suspend/resume.

## Goal

```python
s = Sandbox.create(
    template="base",
    secure=False,
    timeout=60,
    lifecycle={"on_timeout": "pause", "auto_resume": True},
)
time.sleep(65)  # suspended, not deleted
print(s.commands.run("echo back").stdout)  # resumes, prints "back"
s.kill()
```

## In scope

| Area | Deliverable |
|------|-------------|
| Platform | Accept `lifecycle` / `auto_pause` on create; persist `on_timeout=pause` |
| Platform | `POST /sandboxes/{id}/pause`, `POST /sandboxes/{id}/resume` |
| Scheduler | Pause path: Runtime suspend instead of delete |
| Router | Traffic to suspended actor triggers Runtime resume |
| Store | Status `paused` vs `running`; `lifecycle` on GET |

## Out of scope

- `connect`, filesystem, secure tokens
- Custom templates

## API

| Method | Path |
|--------|------|
| POST | `/sandboxes/{sandboxID}/pause` |
| POST | `/sandboxes/{sandboxID}/resume` |

Extend `POST /sandboxes` body: `auto_pause`, `lifecycle`.

**REST parity:** +2 ops → 9/56 (16%). See [e2b-rest-parity.md](e2b-rest-parity.md).
