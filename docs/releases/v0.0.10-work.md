# v0.0.10 — Work breakdown

Implementation sub-problems for [v0.0.10](v0.0.10.md). Five new Platform **read-only** template routes backed by **pre-provisioned** cluster `ActorTemplate` CRs (not user template builds).

**No Substrate code changes** — catalog reads Kubernetes `ActorTemplate` objects; sandbox create still uses existing `CreateActor` + `ResumeActor`.

**Full-field rule:** each shipped operation includes **all** OpenAPI request/response fields. See [e2b-rest-parity.md](e2b-rest-parity.md).

## WP1 — Template catalog (ActorTemplate → E2B shapes)

- Package: `internal/platform/template_catalog.go` (or `internal/template/`) with a small interface:
  - `List(ctx) ([]CatalogTemplate, error)`
  - `Get(ctx, templateID string) (CatalogTemplate, error)`
  - `ResolveAlias(ctx, alias string) (CatalogTemplate, error)`
- Source: in-cluster K8s client listing `ActorTemplate` in `ACTORDOCK_TEMPLATE_NAMESPACE` (default `actordock`)
  - Include only templates with `status.phase=Ready` (match Substrate `PhaseReady`)
  - Fallback for unit tests: static catalog with configured `base` when K8s client nil (httptest)
- Map each CR to internal model with fields needed for E2B JSON:
  - `templateID`: CR name (e.g. `base`) — **same id** used in `POST /sandboxes` `templateID`
  - `names`: `[name]`; `aliases`: `[name]` for v0.0.10 (alias == name)
  - `cpuCount`, `memoryMB`, `diskSizeMB`: from Platform defaults / env (`ACTORDOCK_DEFAULT_*`) or fixed self-hosted defaults documented in code
  - `envdVersion`: `ACTORDOCK_ENVD_VERSION`
  - `buildID`: stable synthetic UUID per template (e.g. derive from namespace+name)
  - Timestamps: CR `creationTimestamp`; `updatedAt` = same or `status` time if available
  - `public`: `true`; `createdBy`: `null`; `lastSpawnedAt`: `null`; `spawnCount` / `buildCount`: `0`
  - `buildStatus`: `ready`
- Unit tests: fake K8s list → mapped catalog; unknown id / alias → `ErrNotFound`

### Deploy / RBAC

- Platform ServiceAccount + Role: `get`, `list`, `watch` on `actortemplates` in template namespace
- Update `manifests/local/platform.yaml` (or separate RBAC manifest in kustomization)

## WP2 — Platform `GET /templates`

- Route: `GET /templates` — optional query `teamID` (accept, ignore for self-hosted)
- **200** JSON array of `Template` — **all 16 required OpenAPI fields** per item
- Empty catalog → **200** `[]`
- Platform unit tests with fake catalog (one `base` entry, field assertions)

## WP3 — Platform `GET /templates/{templateID}`

- Route: `GET /templates/{templateID}` — query `nextToken`, `limit` (parse; pagination optional stretch — document single-page for v0.0.10)
- **200** `TemplateWithBuilds` — all required fields including `builds[]`
- Each `builds[]` entry: synthetic `TemplateBuild` with `buildID`, `status=ready`, `cpuCount`, `memoryMB`, `envdVersion`, timestamps
- Unknown template → **404**
- Platform unit tests

## WP4 — Platform `GET /templates/aliases/{alias}`

- Route: `GET /templates/aliases/{alias}`
- **200** `TemplateAliasResponse` (`templateID`, `public`)
- Unknown alias → **404**
- Platform unit tests (`base` resolves)

## WP5 — Platform `GET /templates/{templateID}/tags`

- Route: `GET /templates/{templateID}/tags`
- **200** JSON array of `TemplateTag` (`tag`, `buildID`, `createdAt`)
- v0.0.10: return **empty array** for known templates (valid full-field response); unknown template → **404**
- Platform unit tests

## WP6 — Platform `GET /templates/{templateID}/files/{hash}`

- Route: `GET /templates/{templateID}/files/{hash}`
- v0.0.10 **stub** for catalog templates only (supports SDK codegen paths; **not** a working build upload):
  - Known template + any hash → **201** `TemplateBuildFileUpload` with `present: true`, `url` omitted or empty (document: build pipeline deferred)
  - Alternative documented default: `present: false` + dummy `url` if SDK requires `url` when not cached — pick one in implementation and test against E2B Python SDK `get_file_upload_link` only if needed
- Unknown template → **404**
- Platform unit tests

## WP7 — Sandbox create template resolution

- Today: `POST /sandboxes` rejects any `templateID` other than configured `ACTORDOCK_TEMPLATE_NAME`
- Change: resolve `templateID` via catalog `Get` (accept id **or** name matching a listed template)
- Map resolved template → `CreateAndResumeSandbox(..., templateNamespace, templateName)` (namespace from config, name = catalog entry)
- Unknown template → **400** (keep current error shape) or **404** — match E2B tests; document choice
- Optional stretch: populate `alias` on sandbox GET/list responses when template has alias (field backfill row in parity tracker)
- Platform unit tests: create with catalog id; reject unknown template

## WP8 — E2E, docs, parity

- E2E (`e2e/tests/test_templates.py`):
  - `GET /templates` returns `base` with required keys
  - `GET /templates/aliases/base` → 200 with `templateID`
  - `GET /templates/{id}/tags` → 200 (empty array OK)
  - Unknown template / alias → 404
  - `Template.exists("base")` via SDK (optional)
- Update `docs/user/quickstart.md`: templates are pre-provisioned; list/get/alias; no user build
- Update [e2b-rest-parity.md](e2b-rest-parity.md): ops #24–25, #27, #38–39 → `done (v0.0.10)`; template GET field rows
- Update [v0.0.10.md](v0.0.10.md) Done when checkboxes

## E2B / SDK reference (this version)

| Capability | v0.0.9 | v0.0.10 |
|------------|--------|---------|
| `GET /templates` | **404** | Lists pre-provisioned templates (`base`, …) |
| `GET /templates/{id}` | **404** | `TemplateWithBuilds` + synthetic `builds[]` |
| `GET /templates/aliases/{alias}` | **404** | Resolves `base` → `templateID` |
| `GET /templates/{id}/tags` | **404** | **200** `[]` or tagged builds (empty OK) |
| `GET /templates/{id}/files/{hash}` | **404** | Stub **201** (no real upload) |
| `Template.build()` / CLI build | **404** | Still **404** / defer |
| `Sandbox.create(template="base")` | Hardcoded name check | Catalog-backed resolution |

**REST parity after v0.0.10:** +5 ops → 27/56 (48%); `alias` / template GET schema backfill (~56% fields est.).

**Semantic note:** E2B cloud templates are **build artifacts**; actordock templates are **cluster ActorTemplates**. JSON shapes match OpenAPI; lifecycle semantics differ (document in quickstart).
