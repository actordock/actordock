# v0.0.3 — Work breakdown

Implementation sub-problems for [v0.0.3](v0.0.3.md). Tackle roughly in order; later items depend on earlier ones.

## WP1 — Store `expires_at` and timeout validation

- Add `expires_at` (`time.Time`) to `store.Sandbox` JSON record
- Helpers for timeout bounds: min (e.g. 15s), max (e.g. 24h), default (300s if omitted on create — per release spec)
- Optional Platform config: `ACTORDOCK_DEFAULT_SANDBOX_TIMEOUT` (seconds)
- Unit tests in `internal/store` and/or `internal/platform`

**E2B semantics:** `timeout` on create and `POST …/timeout` are **seconds from now**, not absolute timestamps.

## WP2 — Create accepts `timeout`

- `POST /sandboxes` body: optional `timeout` (int, seconds) — E2B field name `timeout`
- On create: `expires_at = now + timeout` (default when omitted)
- Reject invalid timeout (`<= 0`, above max) with **400**
- Persist `expires_at` in Redis with existing sandbox record
- Platform unit tests

## WP3 — `POST /sandboxes/{id}/timeout`

- Route: `POST /sandboxes/{id}/timeout`
- Body: `{"timeout": <seconds>}` (`PostSandboxesSandboxIDTimeoutBody`)
- Sets `expires_at = now + timeout` for existing sandbox (SDK `sandbox.set_timeout()`)
- **204** on success; **404** if sandbox missing; **400** for invalid timeout
- Platform unit tests

## WP4 — GET/list expose `endAt`

- Replace hardcoded `defaultSandboxTTL` (24h) in `buildSandboxDetail` / list responses
- `endAt` = `expires_at` from Redis (RFC3339, E2B `SandboxDetail.endAt`)
- GET/list handlers unchanged structurally; wire stored `expires_at`
- Platform unit tests (create + set_timeout reflected in GET)

## WP5 — E2E and documentation

- E2E: `Sandbox.create(..., timeout=60)` → `get_info().end_at` ~60s after `started_at`
- E2E: `set_timeout(120)` extends `end_at` (no auto-kill yet — scheduler is v0.0.4)
- E2E or unit: invalid timeout → 4xx
- Brief `docs/user/quickstart.md` note on timeout
- `hack/verify-local.sh` / CI `install-smoke` runs new tests
- Update `docs/releases/v0.0.3.md` Done when checkboxes when complete

## Out of scope (defer to later versions)

- Scheduler / auto-kill on expiry ([v0.0.4](v0.0.4.md))
- autoPause, lifecycle, `on_timeout`
- Filesystem, secure tokens
- Extending timeout while paused (v0.0.5)

## E2B API reference (this version)

| SDK | HTTP |
|-----|------|
| `Sandbox.create(timeout=…)` | `POST /sandboxes` + `timeout` |
| `sandbox.set_timeout(n)` | `POST /sandboxes/{id}/timeout` + `{"timeout": n}` |
