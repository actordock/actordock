# v0.0.6 — Work breakdown

Implementation sub-problems for [v0.0.6](v0.0.6.md). Tackle roughly in order; later items depend on earlier ones.

**Full-field rule:** each shipped operation includes **all** OpenAPI request/response fields. See [e2b-rest-parity.md](e2b-rest-parity.md). Stub/zero values are OK for logs/metrics when no real telemetry exists yet.

## WP1 — `POST /sandboxes/{id}/refreshes`

- Route: `POST /sandboxes/{sandboxID}/refreshes`
- Body: optional `duration` (seconds); E2B OpenAPI max **3600** (1h) — validate separately from `POST …/timeout` (24h max)
- Sets `expires_at = now + duration`; default `duration` when omitted → `ACTORDOCK_DEFAULT_SANDBOX_TIMEOUT` (same pattern as resume default)
- **204** on success; **404** missing sandbox; **400** invalid duration
- Reuse Redis persist path from `POST …/timeout`
- Platform unit tests

**E2B semantics:** SDK refresh / keep-alive; extends TTL without requiring client `set_timeout`.

## WP2 — Metrics APIs (list + per-sandbox)

- Response types matching OpenAPI: `SandboxMetric` (all 9 fields), `SandboxesWithMetrics` (`sandboxes` map)
- `GET /sandboxes/metrics?sandbox_ids=…` (comma-separated, required; max 100 ids)
- `GET /sandboxes/{sandboxID}/metrics` with optional query `start`, `end` (unix seconds)
- Stub metrics OK: return valid JSON with zero/empty values (no runtime/envd collection in v0.0.6)
- **200** for valid requests; **400** missing/invalid `sandbox_ids`; **404** unknown sandbox on per-id route
- Platform unit tests assert all schema keys present

## WP3 — Logs APIs (v1 + v2)

- Response types: `SandboxLogs` (`logs`, `logEntries`), `SandboxLogsV2Response` (`logs`), `SandboxLog`, `SandboxLogEntry` (incl. `level`, `fields`)
- `GET /sandboxes/{sandboxID}/logs`: query `start`, `limit` (default 1000)
- `GET /v2/sandboxes/{sandboxID}/logs`: query `cursor`, `limit`, `direction`, `level`, `search`
- Accept and parse query params; empty arrays OK (no log shipping from envd in v0.0.6)
- **200** for existing sandbox; **404** missing sandbox
- Platform unit tests per route + field assertions

## WP4 — E2E and documentation

- E2E: at least one metrics call (`GET …/metrics` or list metrics) returns **200** with expected JSON keys
- E2E: at least one logs call (v1 or v2) returns **200** with `logs` / `logEntries` keys
- E2E: `POST …/refreshes` extends `end_at` (like `set_timeout`)
- E2E: unknown sandbox → **404** on per-sandbox logs/metrics/refreshes
- Brief `docs/user/quickstart.md` note on observability routes
- Update `docs/releases/v0.0.6.md` Done when checkboxes; parity tracker ops → `done` for five routes
