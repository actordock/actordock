# v0.0.5

**Status:** planned

## Target

**Idle suspend** — Sandboxes can sleep instead of die; wake on next use.

| Before | After |
|--------|-------|
| Timeout always kills | `on_timeout=pause` suspends actor; next command resumes |

Uses Substrate suspend/resume — no separate pause API yet.

## Goal

```python
s = Sandbox.create(
    template="base",
    secure=False,
    timeout=60,
    lifecycle={"on_timeout": "pause", "auto_resume": True},
)
time.sleep(65)  # suspended, not deleted
print(s.commands.run("echo back").stdout)  # resumes, prints "back"
s.kill()
```

## In scope

| Area | Deliverable |
|------|-------------|
| Platform | Map `lifecycle` / `auto_pause` on create to Redis |
| Scheduler | Pause path: Substrate suspend instead of delete |
| Router | Traffic to suspended actor triggers Substrate resume |
| Store | Status `paused` vs `running` |

## Out of scope

- `POST /sandboxes/{id}/pause` / `connect` (defer post-0.1.0)
- Filesystem, secure tokens
- Custom templates

## API

Extend `POST /sandboxes` body: `auto_pause`, lifecycle fields. No new routes.

## Milestones

1. Scheduler pause path on expiry when `on_timeout=pause`
2. GET shows `paused` after timeout
3. `commands.run` after pause succeeds (resume + execute)
4. Kill still works on paused sandbox
