# v0.0.3

**Status:** planned

## Target

**Manual TTL** — Operator/SDK controls how long a sandbox may live.

| Before | After |
|--------|-------|
| Sandboxes live until killed | `timeout` on create; extend via API; stored in Redis |

## Goal

```python
s = Sandbox.create(template="base", secure=False, timeout=600)
s.set_timeout(900)  # extend
info = Sandbox.get_info(s.sandbox_id)
assert info.end_at > info.started_at
s.kill()
```

## In scope

| Area | Deliverable |
|------|-------------|
| Platform | `POST /sandboxes` accepts `timeout` (seconds) |
| Platform | `POST /sandboxes/{id}/timeout` |
| Store | Redis: `expires_at` per sandbox |
| Platform | GET includes timeout / end_at fields |

## Out of scope

- Background enforcement (scheduler → v0.0.4)
- autoPause, lifecycle
- Filesystem, secure tokens

## API

| Method | Path |
|--------|------|
| POST | `/sandboxes` (body: `timeout`) |
| POST | `/sandboxes/{id}/timeout` |

## Milestones

1. Create with default timeout (e.g. 300s) if omitted
2. POST timeout updates `expires_at` in Redis
3. GET exposes remaining time / end_at
4. Extending timeout on running sandbox succeeds
