# v0.0.9

**Status:** planned

## Target

**Volumes** — E2B volume CRUD on Platform plus **runtime bind-mount** into sandboxes.

| Before | After |
|--------|-------|
| No volume API | Create/list/get/delete volumes via REST |
| `volumeMounts` ignored | Mount validated volumes at sandbox create; files persist across sandboxes |

## Goal

```python
from e2b import Volume, Sandbox

volume = Volume.create("my-data")
sbx = Sandbox.create(
    template="base",
    volume_mounts={"/mnt/data": volume},
)
sbx.commands.run("echo hello > /mnt/data/file.txt")
sbx.kill()

sbx2 = Sandbox.create(template="base", volume_mounts={"/mnt/data": volume})
assert "hello" in sbx2.commands.run("cat /mnt/data/file.txt").stdout
```

## In scope

| Area | Deliverable |
|------|-------------|
| Platform | Full volume CRUD: `GET/POST /volumes`, `GET/DELETE /volumes/{id}` |
| Store | Volume metadata + token + host path in Redis |
| Sandbox API | Accept `volumeMounts` on create; return on get/list |
| Substrate | Per-actor bind mounts in atelet OCI spec (see [v0.0.9-work.md](v0.0.9-work.md) WP5a) |
| Runtime | hostPath volume root on workers; mounts wired at sandbox create |
| Tests | Unit CRUD + E2E cross-sandbox file persistence |

## Out of scope

- GCS/S3 backing (hostPath on worker for local/Kind)
- Standalone volume file SDK when not mounted
- Template file APIs
