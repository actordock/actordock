# v0.0.11

**Status:** planned

## Target

**Ops dashboard (optional)** — Standalone web UI for sandbox visibility; deploy only when needed.

| Before | After |
|--------|-------|
| CLI / curl / SDK only | Browser list + detail for sandboxes, metrics, logs |

Not part of the E2B REST surface. **REST parity unchanged** (still 27/56 after [v0.0.10](v0.0.10.md)).

## Goal

```bash
# Optional — skip in headless / CI clusters
kubectl apply -f manifests/dashboard/
open http://dashboard.localhost
# → sandbox list; click row → metrics snapshot + recent logs
```

## In scope

| Area | Deliverable |
|------|-------------|
| `cmd/dashboard` | Separate binary; talks to Platform REST with `X-API-KEY` |
| UI | Sandbox list (id, state, template, expires); detail page with metrics + logs |
| Deploy | Optional Kubernetes manifest (Deployment + Service); **not** enabled in default install |
| Config | `PLATFORM_URL`, `API_KEY` (env or K8s Secret) |

## Out of scope

- E2B SDK / new Platform API routes
- Write actions (kill, pause, create) — read-only v1
- Auth beyond shared API key; SSO / multi-user
- Historical metrics charts, log export, alerting
- Bundling dashboard into Platform or Router images

## Architecture

```
Browser → dashboard (optional) → Platform REST (existing routes)
Platform / Router / envd — unchanged; dashboard absent if not deployed
```

## Depends on

[v0.0.10](v0.0.10.md) (template metadata nice-to-have; [v0.0.7](v0.0.7.md) real telemetry required for useful logs/metrics panels)
