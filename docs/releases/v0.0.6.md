# v0.0.6

**Status:** planned

## Target

**Observability** — Sandbox logs, metrics, and timeout refresh via E2B REST.

| Before | After |
|--------|-------|
| No logs/metrics endpoints | SDK can query logs/metrics; extend TTL via refreshes |

## Goal

```python
s = Sandbox.create(template="base", secure=False, timeout=60)
# SDK metrics/logs calls return structured (possibly empty) responses
s.set_timeout(120)  # existing
# POST /sandboxes/{id}/refreshes also extends TTL
s.kill()
```

## In scope

| Area | Deliverable |
|------|-------------|
| Platform | `GET /sandboxes/metrics`, `GET /sandboxes/{id}/metrics` |
| Platform | `GET /sandboxes/{id}/logs`, `GET /v2/sandboxes/{id}/logs` |
| Platform | `POST /sandboxes/{id}/refreshes` (extend `expires_at`) |
| E2E | At least one logs/metrics call succeeds without 404 |

## Out of scope

- Real log shipping from envd (stub/empty OK for v0.0.6)
- Prometheus-grade metrics
- Template/volume APIs

## API

| Method | Path |
|--------|------|
| GET | `/sandboxes/metrics` |
| GET | `/sandboxes/{id}/metrics` |
| GET | `/sandboxes/{id}/logs` |
| GET | `/v2/sandboxes/{id}/logs` |
| POST | `/sandboxes/{id}/refreshes` |

**REST parity:** +5 ops → 14/56 (25%). See [e2b-rest-parity.md](e2b-rest-parity.md).

## Depends on

[v0.0.5](v0.0.5.md)

- [x] `refreshes` updates `expires_at` like `set_timeout`
