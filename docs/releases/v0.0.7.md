# v0.0.7

**Status:** released (2026-06-19)

## Target

**Real telemetry** — Sandbox logs and metrics return real data instead of v0.0.6 stubs.

| Before | After |
|--------|-------|
| Logs always empty; metrics zeros / `[]` | `commands.run` output visible in logs; CPU/memory reflect sandbox usage |

No new Platform REST operations (same five observability routes as [v0.0.6](v0.0.6.md)).

## Goal

```python
s = Sandbox.create(template="base", secure=False, timeout=120)
s.commands.run("echo hello")
logs = httpx.get(f"{api}/sandboxes/{s.sandbox_id}/logs", headers=headers).json()
assert any("hello" in e.get("message", "") or "hello" in e.get("line", "") for e in logs["logEntries"] + logs["logs"])

metrics = httpx.get(f"{api}/sandboxes/metrics", params={"sandbox_ids": s.sandbox_id}, headers=headers).json()
m = metrics["sandboxes"][s.sandbox_id]
assert m["memTotal"] > 0
s.kill()
```

## In scope

| Area | Deliverable |
|------|-------------|
| envd | Ring buffer for process stdout/stderr; internal `GET /logs` with query filters |
| envd | Expose cgroup CPU/memory (and disk if feasible) via internal metrics endpoint |
| Platform | `GET …/logs` (v1 + v2): fetch from envd backend; honor `start`/`limit`/`cursor`/filters |
| Platform | `GET …/metrics`: populate `SandboxMetric` from envd (or runtime) instead of stub |
| Platform | `GET …/{id}/metrics`: return time-series or latest samples (non-empty when sandbox ran) |
| E2E | After `commands.run`, logs contain command output; list metrics show non-stub values |

## Out of scope

- Runtime node / pod log aggregation (only envd in-sandbox collection for v0.0.7)
- Prometheus export, historical metrics retention beyond sandbox lifetime
- Log persistence across kill (in-memory only OK)
- `POST …/connect`, network, snapshots ([v0.0.8](v0.0.8.md))
- GET sandbox detail resource fields backfill from live metrics (optional stretch)

## API (unchanged routes; semantics upgrade)

| Method | Path | v0.0.7 change |
|--------|------|---------------|
| GET | `/sandboxes/metrics` | Real `SandboxMetric` values |
| GET | `/sandboxes/{id}/metrics` | Non-empty samples when available |
| GET | `/sandboxes/{id}/logs` | Real `logs` + `logEntries` |
| GET | `/v2/sandboxes/{id}/logs` | Real structured `logs` |

**REST parity:** still **14/56 (25%)** ops; observability routes move from stub semantics to real data. See [e2b-rest-parity.md](e2b-rest-parity.md).
