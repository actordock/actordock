# v0.0.4

**Status:** released (2026-06-19)

## Target

**Auto cleanup** — Expired sandboxes die without a client calling `kill()`.

| Before | After |
|--------|-------|
| Timeout is metadata only | Scheduler kills actor when `expires_at` passes |

## Goal

```python
s = Sandbox.create(template="base", secure=False, timeout=30)
time.sleep(35)
# sandbox gone; GET returns 404
```

```bash
./hack/install-local.sh && ./hack/verify-local.sh
```

Default lifecycle: `on_timeout=kill` (E2B default).

## In scope

| Area | Deliverable |
|------|-------------|
| Scheduler | Single-replica worker; poll Redis for `expires_at < now` |
| Scheduler | Call Runtime delete + purge Redis |
| Platform | Default `lifecycle.on_timeout=kill` on create |

## Out of scope

- `on_timeout=pause` (v0.0.5)
- Explicit pause API, connect
- Filesystem, secure tokens

## API

No new routes. Scheduler is internal.

## Milestones

1. Scheduler deployment in `actordock` namespace
2. Kill path: actor deleted, Redis cleaned, GET → 404
3. Non-expired sandboxes untouched
4. Scheduler restart: no double-delete crashes
5. `./hack/verify-local.sh` covers auto-cleanup E2E
