# v0.0.7 — Work breakdown

Implementation sub-problems for [v0.0.7](v0.0.7.md). Same five Platform observability routes as v0.0.6; replace stub payloads with real envd-sourced data.

## WP1 — envd log collection

- In-memory ring buffer (cap e.g. 10k lines or 4 MiB) inside envd process
- On `process.Start`: append stdout/stderr lines with timestamp, level (`info`/`error`), optional `fields` (e.g. `stream=stdout`)
- Optional: capture envd own slog via custom handler (stretch)
- Unit tests: write lines, query buffer with filters

**E2B mapping:** feeds both `SandboxLog` (`line`) and `SandboxLogEntry` (`message`, `level`, `fields`).

## WP2 — envd logs HTTP API

- Internal route: `GET /logs` (envd only, not Platform)
- Query params aligned with Platform: `start`, `limit`, `cursor`, `direction`, `level`, `search`
- JSON response: structured entries Platform can map to v1/v2 schemas
- Unit tests per filter; invalid query → 400

## WP3 — Platform logs from envd

- Replace `buildStubSandboxLogs()` / `buildStubSandboxLogsV2()` with envd fetch via `GetActorBackend`
- Apply existing Platform query validation; forward params to envd
- Sandbox missing → **404**; envd unreachable → **200** empty arrays (or **502** — pick one, document)
- Unit tests with httptest envd stub returning sample logs
- Extend E2E: `commands.run("echo hello")` then GET logs v1/v2 contains `hello`

## WP4 — envd metrics collection

- Read cgroup stats for sandbox container (CPU %, memory used/total, disk if available)
- Internal route: `GET /metrics` (latest snapshot) and optional `GET /metrics/history?start=&end=` for per-id Platform route
- Unit tests with fake cgroup reader or test fixture

## WP5 — Platform metrics from envd

- Replace `buildStubSandboxMetric` / empty `[]` per-id response with envd (or cached) samples
- `GET /sandboxes/metrics`: one current `SandboxMetric` per requested id
- `GET /sandboxes/{id}/metrics`: array of samples in `[start, end]` when query present; else latest or `[latest]`
- Unit + E2E: after sandbox activity, `cpuUsedPct` or `memUsed` reflects non-zero usage

## WP6 — E2E, docs, parity

- Update `e2e/tests/test_observability.py`: assert non-empty logs after command; metrics not all zeros
- Update `docs/user/quickstart.md`: note real telemetry (remove stub disclaimer)
- Update [e2b-rest-parity.md](e2b-rest-parity.md): logs/metrics field rows — real data, not stub OK
- Update [v0.0.7.md](v0.0.7.md) Done when checkboxes
