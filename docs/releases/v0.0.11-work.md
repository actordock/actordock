# v0.0.11 — Work breakdown

Implementation sub-problems for [v0.0.11](v0.0.11.md). Standalone **Actordock** dashboard (reference: [e2b-dev/dashboard](https://github.com/e2b-dev/dashboard) information architecture; visual language: **Greek marble sculpture**).

**No new Platform REST routes** — dashboard consumes existing Platform APIs via `X-API-KEY`. Optional **BFF proxy** in `dashboard/` server to avoid browser CORS and keep the API key server-side.

**Component layout:** all dashboard code lives under one top-level **`dashboard/`** directory (Go server + React SPA). K8s manifests stay in `manifests/dashboard/` (repo deploy convention). Root `Makefile` / `ko` build `./dashboard/cmd/dashboard`.

```
dashboard/                    # self-contained optional component
  README.md
  cmd/dashboard/main.go       # thin entrypoint
  internal/server/            # config, Platform proxy, static/embed
  web/                        # Vite + React + TypeScript SPA
    src/
    package.json
    dist/                     # production build (go:embed)
manifests/dashboard/          # optional K8s Deployment + Service (not default install)
```

**Scope tiers (all in v0.0.11):**

| Tier | Pages |
|------|-------|
| **P0** | Settings, Overview, Sandbox list, Sandbox detail (Overview / Metrics / Logs) |
| **P1** | Templates list + detail, Volumes list + detail, Snapshots list |
| **P2** | Sandbox write actions, Terminal, aggregate Monitoring, metrics sparklines, log export |

## Design system — Greek marble sculpture (WP0)

Apply across all pages before feature pages ship.

### Visual language

| Token | Value | Use |
|-------|-------|-----|
| `--marble-base` | `#F5F0E8` | Page background (warm marble) |
| `--marble-card` | `#FAF7F2` | Card / panel surface |
| `--marble-shadow` | `#E8E2D8` | Borders, dividers |
| `--stone-ink` | `#2C2A26` | Primary text |
| `--stone-muted` | `#8A8680` | Secondary text |
| `--bronze-accent` | `#B8956A` | Links, active nav, primary buttons |
| `--bronze-deep` | `#8A6F4A` | Hover / pressed |
| `--laurel-ok` | `#6B7F5E` | Running / success |
| `--amber-pause` | `#C4A574` | Paused / warning |
| `--terracotta-danger` | `#A65D4E` | Error / kill |

### Typography

- **Display / headings:** Cinzel or Playfair Display (classical serif)
- **UI / body:** Inter or Source Sans 3
- **Monospace (ids, logs):** JetBrains Mono

### Layout motifs

- **Column nav:** fixed left rail (~240px), vertical labels like a temple colonnade; active item = bronze left border + subtle marble inset
- **Entablature header:** thin top bar with product name **Actordock** + connection status pill
- **Exhibit cards:** content in rounded rectangles with 1px `--marble-shadow` border and soft drop shadow (no heavy glassmorphism)
- **Background:** very low-opacity Greek column / sculpture line art (SVG), never behind text at >8% opacity
- **State badges:** pill with classical thin border; icons linear only (torch = running, broken column = failed, sleeping muse = paused)

### Motion

- Page enter: 150ms fade + 4px upward translate
- Refresh: subtle shimmer on metric numbers (marble light catch)
- No bounce / neon animations

### Deliverables (WP0)

- `dashboard/web/` — Vite + React + TypeScript SPA
- `dashboard/web/src/styles/theme.css` — CSS variables above
- Shared layout: `AppShell`, `SideNav`, `PageHeader`, `MarbleCard`, `StatusBadge`, `DataTable`
- Storybook optional; at minimum one `ThemePreview` route for design QA
- Favicon: simplified Ionic column capital (SVG) under `dashboard/web/public/`

## WP1 — Dashboard server (`dashboard/`)

- Entry: `dashboard/cmd/dashboard/main.go`
- Go packages: `dashboard/internal/server/` — config, proxy handler, static file server, `go:embed` of `dashboard/web/dist`
- Config (env):
  - `ACTORDOCK_PLATFORM_URL` (default `http://platform:8080` in-cluster)
  - `ACTORDOCK_API_KEY` (required in prod; dev default `dev` documented)
  - `ACTORDOCK_DASHBOARD_ADDR` (default `:3000`)
  - `ACTORDOCK_DASHBOARD_PROXY_PLATFORM` (default `true` — server-side proxy)
- Behaviors:
  - `GET /health` — dashboard liveness
  - `GET /api/platform/*` — reverse proxy to Platform with injected `X-API-KEY` (strip path prefix)
  - `GET /*` — serve embedded SPA (`go:embed web/dist`)
- Build:
  - `cd dashboard/web && pnpm install && pnpm build` → `web/dist`
  - `go build -o ../../bin/dashboard ./dashboard/cmd/dashboard` (or `make build-dashboard`)
  - `ko build ./dashboard/cmd/dashboard` for container image
- Unit tests: `dashboard/internal/server/*_test.go` — proxy forwards headers; missing API key → 503 on `/api/platform/health`

## WP2 — Settings & connection (P0)

**Route:** `/settings`

- Form: Platform URL (when not fully env-driven), API key (optional override; prefer server env in K8s)
- **Test connection:** `GET /api/platform/health` + `GET /api/platform/sandboxes?limit=1`
- Persist UI prefs in `localStorage`: refresh interval, log tail lines, theme density
- Show read-only cluster hints if env provides `ACTORDOCK_VERSION` (optional stretch)
- Error states in terracotta panel with plain-language fix steps

## WP3 — Overview home (P0)

**Route:** `/`

- Cards (marble exhibit style):
  - Total sandboxes / by state (`running`, `paused`, …) from `GET /sandboxes`
  - Platform reachable (health)
  - Templates count (`GET /templates`)
  - Volumes count (`GET /volumes`)
- **Recent sandboxes** table (last 5 by `startedAt`)
- Quick links to Sandboxes / Templates / Volumes / Snapshots
- Auto-refresh every 30s (configurable)

## WP4 — Sandbox list (P0)

**Route:** `/sandboxes`

- Data: `GET /sandboxes` or `GET /v2/sandboxes`
- Table columns: sandbox id (mono, copy), state badge, template, alias, CPU/MEM (from batch metrics), startedAt, endAt
- Batch metrics: `GET /sandboxes/metrics` merged into rows
- Filters: state multi-select, template dropdown, text search on id
- Sort: startedAt, endAt, state
- Row click → `/sandboxes/:id`
- Empty state: sculpture silhouette + link to **Create sandbox** (P2 action entry)

---

## WP5 — Sandbox detail — Overview tab (P0)

**Route:** `/sandboxes/:id` (default tab)

- Data: `GET /sandboxes/:id`
- Sections: Identity, Resources, Lifecycle (`onTimeout`, `autoResume`), Network, Volume mounts, Template link
- **Action bar placeholder** (wired in WP11): Kill, Pause, Resume, Extend timeout
- Breadcrumb: Sandboxes → `{id}`

---

## WP6 — Sandbox detail — Metrics tab (P0 + P2 sparklines)

**Route:** `/sandboxes/:id/metrics`

- Snapshot: `GET /sandboxes/:id/metrics` — CPU, memory, disk gauges (bronze arc or horizontal bars)
- Manual refresh + auto-refresh (5s / 15s / off)
- **P2:** client-side ring buffer (last N polls) → sparkline per metric (no new backend)
- Link to aggregate **Monitoring** page (WP13)

## WP7 — Sandbox detail — Logs tab (P0 + P2 export)

**Route:** `/sandboxes/:id/logs`

- Data: `GET /v2/sandboxes/:id/logs` (prefer v2 structured entries)
- Virtualized list: timestamp, level, command, stdout/stderr snippet
- Filters: level, text search
- Tail / pause / auto-scroll
- **P2:** Download as `.txt` or `.jsonl` (client-side export of fetched entries)

## WP8 — Templates (P1)

**Routes:** `/templates`, `/templates/:id`

- List: `GET /templates` — templateID, buildStatus, resources, envdVersion, public
- Detail: `GET /templates/:id` — builds array, aliases, timestamps
- Sub-sections (tabs): Overview | Builds | Tags (`GET /templates/:id/tags`)
- Read-only; **no** build/upload buttons (Substrate backlog [#59](https://github.com/actordock/actordock/issues/59))
- Link from sandbox overview → template detail

## WP9 — Volumes (P1)

**Routes:** `/volumes`, `/volumes/:id`

- List: `GET /volumes` — volumeID, name, createdAt
- Detail: `GET /volumes/:id` — token (mask/reveal toggle), hostPath (if exposed)
- **Mounted by:** derive from `GET /sandboxes` → filter `volumeMounts` containing this volume name (client-side join)
- Note banner: runtime bind-mount backlog [#48](https://github.com/actordock/actordock/issues/48) / [#49](https://github.com/actordock/actordock/issues/49)

## WP10 — Snapshots (P1)

**Route:** `/snapshots`

- List: `GET /snapshots` — snapshot id, sandbox id, timestamps
- Row → sandbox detail if sandbox still listed; else show orphaned snapshot metadata
- Read-only; create snapshot action deferred to WP11 (uses existing `POST /sandboxes/:id/snapshots`)

## WP11 — Sandbox write actions (P2)

Dashboard BFF or direct proxy calls to existing Platform routes:

| Action | Platform API |
|--------|----------------|
| Create sandbox | `POST /sandboxes` (modal: template, timeout, volumeMounts JSON) |
| Kill | `DELETE /sandboxes/:id` |
| Pause | `POST /sandboxes/:id/pause` |
| Resume | `POST /sandboxes/:id/resume` |
| Set timeout | `POST /sandboxes/:id/timeout` |
| Refresh TTL | `POST /sandboxes/:id/refreshes` |
| Create snapshot | `POST /sandboxes/:id/snapshots` |
| Network | `PUT /sandboxes/:id/network` (advanced drawer) |

- Confirm dialogs for destructive actions (kill) — classical modal with bronze primary
- Toast notifications (marble snackbar, bronze border)
- Optimistic UI optional; always reconcile on error
- Unit/integration tests against httptest Platform or mock fetch layer

---

## WP12 — Terminal tab (P2)

**Route:** `/sandboxes/:id/terminal`

- Flow:
  1. `POST /sandboxes/:id/connect` (extend TTL if needed)
  2. Open interactive session via Router/envd Connect (same path E2B uses for command streaming)
- Implementation options (pick in PR, document choice):
  - **A:** xterm.js + dashboard WebSocket proxy to Router Connect RPC
  - **B:** embed `@connectrpc/connect-web` in browser with signed connect token from Platform response
- Only enabled when sandbox `state == running` (or auto-resume on connect per Platform behavior)
- Fallback: show connect URL + copy button if full PTY proxy slips schedule

---

## WP13 — Monitoring aggregate (P2)

**Route:** `/sandboxes/monitoring`

- E2B analogue: sandboxes monitoring tab
- `GET /sandboxes/metrics` — sortable table of all sandboxes with CPU/MEM/disk
- Highlight hot sandboxes (>80% CPU or memory)
- Auto-refresh; links to per-sandbox metrics tab

- Bundling dashboard into Platform or Router images
- Alerting / PagerDuty integrations
