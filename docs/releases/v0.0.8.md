# v0.0.8

**Status:** released (2026-06-21)

## Target

**Sandbox extras (real)** — Connect, network, and snapshots with working behavior, not schema-only stubs.

| Before | After |
|--------|-------|
| No connect/network/snapshot REST | SDK paths work **and** underlying behavior is real |

## Goal

```python
s = Sandbox.create(template="base", secure=False, timeout=120)
# Interactive connect: open terminal session via connect URL / SDK
s.commands.run("echo connect-ok")

# Network: disable internet → outbound fetch fails (or succeeds when enabled)
# s.network...  (SDK)

snap = s.snapshot.create()  # triggers real checkpoint; listed in GET /snapshots
assert snap.snapshot_id
s.kill()
```

## In scope

| Area | Deliverable |
|------|-------------|
| envd | `process.Connect`: PTY + bidirectional stream (replace Unimplemented) |
| Router | Proxy connect traffic to envd; enforce `allow_internet_access` on egress |
| Platform | `POST …/connect` — resume if needed, return usable connect endpoint |
| Platform | `PUT …/network` — persist; `GET …/{id}` reflects network fields |
| Platform + runtime | `POST …/snapshots` — checkpoint/suspend actor, store metadata in Redis |
| Platform | `GET /snapshots` — list persisted snapshot records |
| E2E | Prove connect session, network toggle, snapshot create + list |

## Out of scope

- Snapshot restore into new sandbox / template build from snapshot
- Full L4/L7 firewall product (minimal egress allow/deny only)
- `POST /sandboxes` network field on create (optional stretch; PUT is required)

## API

| Method | Path | v0.0.8 real behavior |
|--------|------|----------------------|
| POST | `/sandboxes/{sandboxID}/connect` | Returns endpoint; envd Connect stream works via Router |
| PUT | `/sandboxes/{sandboxID}/network` | Stored + enforced on Router egress |
| POST | `/sandboxes/{sandboxID}/snapshots` | runtime checkpoint + Redis metadata |
| GET | `/snapshots` | Lists created snapshots |
